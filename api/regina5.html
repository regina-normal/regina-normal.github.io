Regina - Moving from 4.x to 5.x

This page relates only to C++ and Python programmers.

Regina has undergone major changes to its C++ and Python interfaces. These changes happened in two stages:

  1. Over many years, Regina's classes and routines were simplified and improved in many ways, and much of its old code became unnecessary. Nevertheless, the old classes and routines were kept around until Regina 5.0 so that old scripts would still work. All of this old code was removed in Regina 5.1.
  2. A second (much simpler) round of changes then took place, mostly to simplify class names and C++ header locations. These changes were implemented in Regina 5.1. Again the old class names and headers were kept for backward compatibility. All of this old code was removed in Regina 6.1.

How do I fix my code?

  • You must incorporate the first-round changes now if you wish your code to work with Regina 5.1 and above. Details of these changes are described below.
  • You must incorporate the second-round changes now if you wish your code to work with Regina 6.1 and above. See the separate page on moving from Regina 5.x to 6.x for details.

First-round changes (Regina 4.x to 5.x)

There are some general rules that cover many of the changes:

  • String output routines of the form x.toString() and x.toStringLong() have been renamed to x.str() and x.detail(). Moreover, in Python you can just print x.
  • Query routines of the form x.getFoo() have been renamed to x.foo().
  • Size routines of the form x.getNumberOfFoo() have almost all been renamed to x.countFoo(). The exception is that some have been renamed to x.size() where this makes sense; all such exceptions are listed in the table below.
  • Template implementation headers of the form foo.tcc have been renamed to foo-impl.h.

Beyond these general rules, many other routines and classes will be (or already have been) removed. These are listed in the table below, along with suggested replacements that you can use instead. Any changes in the behaviour of these routines or classes are noted in the comments column.
