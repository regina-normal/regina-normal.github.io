Regina for Debian: Bullseye (Debian 11.x)

Debian 11.x (Bullseye) already comes with Regina, but this will be forever fixed at version 5.1.

To keep up-to-date with the newest version of Regina, follow these instructions.

You only need to follow these steps the first time you install Regina. After this, Debian's package manager will handle upgrades for you, just like for any other Debian package.

  1. Open a terminal and grant yourself root privileges.

  2. Install the debian-keyring package. This will give you Regina's cryptographic signing key (since Ben is a Debian developer). To do this, type:

    apt-get install debian-keyring

  3. Add Regina's location to your list of repositories. To do this, download regina-bullseye.sources from here and copy it into the directory /etc/apt/sources.list.d/ (in the line below, replace ~/Downloads with your real download location):

    cp -i ~/Downloads/regina-bullseye.sources /etc/apt/sources.list.d/

    Note the suffix .d on the directory name! You do not want to overwrite sources.list.

    This file /etc/apt/sources.list.d/regina-bullseye.sources is just a short text file; its full contents are included at the bottom of this page.

  4. Update the list of available packages. To do this, type:

    apt-get update

    If you see an error like “the method driver /usr/lib/apt/methods/https could not be found”, install the package apt-transport-https and try again.

    If you see an error like “the public key is not available”, it is possibly because you forgot to install debian-keyring in the first step.

  5. Install or upgrade Regina like you would any other package. Type:

    apt-get install regina-normal

This procedure only installs the main packages regina-normal and regina-normal-doc, which is all that most people will need.

Some tips:

  • If you are a C++ programmer and you wish to build Regina into your own projects, you should also install the package regina-normal-dev.

  • If you had previously added Regina's repository as a single line in /etc/apt/sources.list, you can comment that line out now (or just delete it). The new file regina-bullseye.sources encodes the same repository but with better security. (The old apt-line will still work, so there is no urgency to change.)

For reference, the file /etc/apt/sources.list.d/regina-bullseye.sources should contain the following text (and the spaces on the last two lines are important): 

X-Repolib-Name: Regina
Types: deb deb-src
URIs: https://people.debian.org/~bab/regina
Suites: bullseye/
Signed-By:
  /usr/share/keyrings/debian-keyring.gpg
  519A0009FB50255DDB4E889270A6BEDF542D38D9
